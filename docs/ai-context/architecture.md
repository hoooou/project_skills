# 架构

## 高层架构概览

该项目采用**文档驱动架构**,是一个纯文档型项目库,包含一组独立的 Claude Code Skills。

```text
概念层级:
┌─────────────────────────────────────────┐
│   Skills 库 (本项目)                     │
│                                         │
│  ┌─────────────┬─────────────┬────────┐│
│  │ project-init│project-review│project-│││
│  │             │              │ sync   ││
│  └──────┬──────┴───────┬──────┴────┬───┘│
│         │              │           │    │
│    ┌────▼──────┐  ┌───▼────┐  ┌──▼───┐ │
│    │中文版 SKILL│  │中文版   │  │中文版 │ │
│    │.md        │  │SKILL.md│  │SKILL.│ │
│    └───────────┘  └────────┘  └──────┘ │
│    ┌───────────┐  ┌────────┐  ┌──────┐ │
│    │英文版-en  │  │英文版-en│  │英文版│ │
│    │SKILL.md   │  │SKILL.md│  │-en   │ │
│    └───────────┘  └────────┘  └──SKILL│ │
└─────────────────────────────────.md───┘│
                                 └───────┘
```

## 主要组件

### 技能定义文件 (SKILL.md)
- **职责**: 定义单个技能的完整行为规范
- **结构**: YAML frontmatter + Markdown 正文
- **所在位置**: 每个技能目录下
- **关键字段**:
  - `name`: 技能标识符 (必须与目录名匹配)
  - `version`: 语义化版本号
  - `description`: 500 字符内的用途描述

### 技能目录
- **职责**: 组织和隔离单个技能的所有文件
- **命名规范**: `<skill-name>/` (中文版) 或 `<skill-name-en>/` (英文版)
- **当前技能**:
  - project-init (初始化)
  - project-review (审查)
  - project-sync (同步)

## 层级和依赖

该项目没有传统意义的技术层级,因为它是纯文档项目。但存在**概念依赖层级**:

```text
依赖方向 (从下到上):
┌────────────────────────────────┐
│   用户项目 (被管理的目标项目)    │ ← 依赖
└───────────┬────────────────────┘
            │ 被这些技能操作
┌───────────▼────────────────────┐
│   Skills (本项目)               │
│   - project-init               │
│   - project-review             │
│   - project-sync               │
└───────────┬────────────────────┘
            │ 被加载和执行
┌───────────▼────────────────────┐
│   Claude Code 运行时环境        │
│   (CLI / Desktop / Web)        │
└────────────────────────────────┘
```

**技能之间的关系**:
- `project-init` → 独立使用,但其输出被其他技能依赖
- `project-sync` → 依赖 `project-init` 建立的知识库结构
- `project-review` → 依赖 `project-init` 建立的知识库结构

**跨技能禁止依赖**:
- 每个技能应自包含,不依赖其他技能的内部实现细节
- 仅依赖共同约定的知识库结构 (AGENTS.md, docs/ai-context/)

## 运行时形态

**部署形态**: Skills 以文件系统形式存在,被 Claude Code 按需加载

**执行模式**:
1. 用户调用技能 (通过 CLI 或界面)
2. Claude Code 读取对应 `SKILL.md`
3. Claude 根据技能指令操作用户项目
4. 技能执行完成,控制权返回用户

**无需运行时依赖**:
- 无需编译或构建
- 无需外部服务
- 无需数据库或持久化存储
- 无需网络连接

## 架构约束

1. **自包含原则**: 每个 `SKILL.md` 必须是完整的技能定义,不依赖外部文件
2. **版本独立性**: 中英文版本应独立维护版本号,但建议同步
3. **向后兼容**: 技能升级应避免破坏已有项目的知识库结构
4. **语言隔离**: 中文版和英文版应在不同目录,避免混淆
5. **元数据严格性**: YAML frontmatter 必须符合 Claude Code 技能规范
