# API

## Not applicable

该项目是纯文档型 Claude Code Skills 库,不提供或使用 API。

## 理由

1. **项目类型**: 静态技能定义文档库
2. **无服务组件**: 不运行服务器或后台服务
3. **无网络通信**: 技能执行过程不进行网络调用
4. **本地文件操作**: 所有操作都在本地文件系统

## 技能调用机制

虽然本项目不提供 API,但技能被 Claude Code 调用时存在**调用接口**:

**输入接口**:
- 技能名称 (如 `project-init`)
- 用户项目路径
- 用户提供的参数或上下文

**输出接口**:
- 执行结果文本
- 生成或更新的文件
- 错误信息 (如果失败)

但这不是传统意义的 REST API、GraphQL API 或 RPC 接口,而是**文件系统级别的接口约定**。

## 目标项目的 API

需要注意区分:
- **本项目 (skills 库)**: 无 API
- **目标项目 (被技能操作的项目)**: 可能有 API

`project-init`、`project-sync` 和 `project-review` 技能会分析和记录**目标项目**的 API 信息,但这不意味着 skills 库本身提供 API。
