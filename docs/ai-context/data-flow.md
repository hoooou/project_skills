# 数据流

## 主要数据流

该项目是纯文档型项目,没有传统意义的运行时数据流。但存在**文档生命周期数据流**和**技能执行数据流**。

## 1. 文档生命周期数据流

```text
技能定义维护流:

┌──────────────┐
│  开发者编辑   │
│  SKILL.md    │
└──────┬───────┘
       │
       ▼
┌──────────────────────┐
│  Git commit & push   │
└──────────┬───────────┘
           │
           ▼
┌────────────────────────┐
│  用户安装/更新 Skills  │
└──────────┬─────────────┘
           │
           ▼
┌────────────────────────┐
│  Claude Code 加载技能   │
└────────────────────────┘
```

**输入边界**: 开发者编辑 SKILL.md 文件
**输出边界**: Claude Code 运行时环境加载技能
**中间传递**: Git 版本控制系统

## 2. 技能执行数据流 (project-init)

```text
初始化技能执行流:

用户调用          Claude Code         目标项目
   │                  │                  │
   │─── /project-init ──→                │
   │                  │                  │
   │                  │───── 读取项目 ────→│
   │                  │      (结构/代码)  │
   │                  │←──── 项目信息 ────│
   │                  │                  │
   │                  │  分析 & 生成文档  │
   │                  │                  │
   │                  │─── 写入 AGENTS.md ──→│
   │                  │─── 写入 docs/ai-context/ ──→│
   │                  │                  │
   │←─── 完成报告 ──────│                  │
   │                  │                  │
```

**输入**:
- 用户项目的文件结构
- 现有代码和文档
- 配置文件 (package.json, pom.xml, etc.)
- 用户提供的补充信息

**输出**:
- `AGENTS.md`: 项目工作协议
- `docs/ai-context/architecture.md`: 架构文档
- `docs/ai-context/modules.md`: 模块文档
- `docs/ai-context/data-flow.md`: 数据流文档
- `docs/ai-context/database.md`: 数据库文档
- `docs/ai-context/api.md`: API 文档
- `docs/ai-context/features.md`: 功能想法
- `docs/ai-context/decisions/`: 决策记录目录

**处理过程**:
1. 只读探索项目
2. 理解技术栈和结构
3. 识别模块和关系
4. 生成结构化文档
5. 记录不确定性

## 3. 技能执行数据流 (project-sync)

```text
同步技能执行流:

用户调用          Claude Code         目标项目        Git
   │                  │                  │           │
   │─── /project-sync ──→                │           │
   │                  │                  │           │
   │                  │───────── git diff ──────────→│
   │                  │←──────── 变更集 ─────────────│
   │                  │                  │           │
   │                  │─── 读取受影响文件 ────→│
   │                  │←─── 文件内容 ──────────│
   │                  │                  │
   │                  │─── 读取现有文档 ──────→│
   │                  │←─── 文档内容 ──────────│
   │                  │                  │
   │                  │  分析影响范围 & 更新  │
   │                  │                  │
   │                  │─── 更新文档 ─────────→│
   │                  │                  │
   │←─── 同步报告 ──────│                  │
   │                  │                  │
```

**输入**:
- Git diff (staged 和 unstaged 变更)
- 变更文件的内容
- 现有的项目文档
- Commit 历史 (可选)

**输出**:
- 更新后的受影响文档
- 新增或更新的决策记录
- 技术债提醒
- 同步摘要报告

**处理过程**:
1. 分析变更集
2. 判断影响范围
3. 读取受影响的文档
4. 对比变更与文档
5. 更新受影响的文档部分
6. 记录新的决策或变更

## 4. 技能执行数据流 (project-review)

```text
审查技能执行流:

用户调用          Claude Code         目标项目
   │                  │                  │
   │─── /project-review ──→              │
   │                  │                  │
   │                  │─── 读取所有文档 ────→│
   │                  │←─── 文档内容 ───────│
   │                  │                  │
   │                  │─── 读取代码 ────────→│
   │                  │←─── 代码内容 ───────│
   │                  │                  │
   │                  │  对比 & 分析健康度  │
   │                  │                  │
   │←─── 健康报告 ──────│                  │
   │  (包含建议分类)    │                  │
   │                  │                  │
```

**输入**:
- 所有项目文档 (AGENTS.md, docs/ai-context/)
- 项目代码
- Git 历史 (可选)
- 测试代码

**输出**:
- 项目知识健康度报告
- 文档漂移分析
- 缺失项清单
- 分类建议 (AI-safe / 需批准 / 需人决策)

**处理过程**:
1. 盘点项目知识
2. 对比文档与代码
3. 检查 AGENTS.md 质量
4. 评估 features.md 有效性
5. 识别技术债
6. 分类建议并生成报告

## 跨模块数据交互

```text
三个技能之间的数据流:

project-init
     │ 生成
     ▼
┌────────────────┐
│  项目知识库     │ ◄─── project-sync 更新
│  (AGENTS.md +  │
│   ai-context)  │
└────────────────┘
     │ 被审查
     ▼
project-review
     │ 发现问题
     ▼
   反馈到
project-sync
```

## 外部集成

**Claude Code 运行时环境**:
- **输入**: 技能调用请求 + 用户项目路径
- **输出**: 执行结果 + 生成的文档

**Git 版本控制**:
- **输入**: git diff, git status, git log 查询
- **输出**: 变更信息、文件状态、提交历史

**文件系统**:
- **输入**: 读取项目文件、现有文档
- **输出**: 写入新文档、更新现有文档
