# 决策记录

本目录用于记录已经由人确认或由项目文件明确体现的重要决策。

## 记录原则

- 不把 AI 推测写成已确认决策。
- 如果只发现问题或机会，先记录为开放问题或建议。
- 决策记录应尽量简短，并包含：
  - 背景
  - 已确认决策
  - 影响
  - 未知项或后续问题

## 当前已确认事实

- 项目以独立目录维护多个 skill。
- 每个 skill 的入口文件是 `SKILL.md`。
- 当前存在中文和英文两套 skill。
- `docs/ai-context/features.md` 用于记录候选需求、功能和优化点，不等同于路线图或已确认决策。

## 尚未记录为决策的开放问题

- 是否需要根目录 README。
- 是否需要自动校验 skill 文档结构。
- 是否需要正式定义中英文同步策略。
- 是否需要记录安装、发布或版本流程。
- 是否需要为 `features.md` 定义条目状态、归档规则或清理周期。
