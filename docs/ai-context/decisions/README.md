# 决策记录目录

本目录用于记录 Skills 项目的重要架构决策和设计选择。

## 如何记录决策

每个决策应创建独立的 Markdown 文件,命名格式:
```
YYYY-MM-DD-<decision-title>.md
```

例如:
- `2026-07-07-use-separate-dirs-for-language-versions.md`
- `2026-07-07-yaml-frontmatter-for-metadata.md`

## 决策记录模板

```markdown
# <决策标题>

## 状态
[Proposed | Accepted | Deprecated | Superseded]

## 上下文
<描述做出决策时的情况、约束和需求>

## 决策
<简洁描述最终决定>

## 理由
<解释为什么做出这个决策,考虑了哪些备选方案>

## 后果
### 优点
- <积极影响>

### 缺点
- <负面影响或权衡>

### 风险
- <潜在风险及缓解措施>

## 替代方案
<列出考虑过但未采纳的其他方案>

## 相关决策
- <链接到相关决策记录>
```

## 当前已知决策 (待记录)

基于项目分析,以下决策可能值得正式记录:

1. **使用独立目录存放不同语言版本**
   - 决策: 中文版和英文版放在不同目录,而不是在同一目录用不同文件名
   - 理由: 避免混淆,清晰隔离

2. **使用 YAML frontmatter 存储技能元数据**
   - 决策: 在 SKILL.md 顶部用 YAML 块定义 name/version/description
   - 理由: 结构化、易解析、符合 Claude Code 规范

3. **技能自包含原则**
   - 决策: 每个 SKILL.md 必须完整,不依赖外部文件
   - 理由: 简化部署和理解,提高可移植性

4. **三技能闭环工作流**
   - 决策: 提供 init/sync/review 三个协同但独立的技能
   - 理由: 覆盖项目知识库的完整生命周期

这些决策目前隐含在技能定义和项目结构中,建议在未来正式记录。
